# Changelog

All notable changes to this project are documented here. Generated by
`aicommit changelog` (the conventional-commit fast path) and then folded
into per-version sections by hand.

Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/);
the project follows [Semantic Versioning](https://semver.org/).

## [1.0.0] ā€” 2025-12-18

### Added
- `pipx install aicommit` packaging metadata and console-script entries
  for both `aicommit` and `git-aicommit`.
- GitHub Actions CI: ruff + mypy + pytest matrix (py3.11 / 3.12 / 3.13)
  plus an sdist+wheel build step.
- `CHANGELOG.md` itself ā€” generated by aicommit, polished by hand.

### Changed
- Polished README with the `pipx install` line and the asciinema-placeholder
  demo block.

## [0.6.0] ā€” 2025-12-15

### Added
- Optional **llama-cpp-python** backend (`pip install 'aicommit[llama-cpp]'`).
- **Streaming** output in `--print` / piped mode via Ollama's NDJSON stream.
- **Token-budget aware diff truncation** at file boundaries.
- Unit-test suite (36 tests) with a tiny VCR-style cassette harness that
  replays recorded Ollama JSON transcripts.

### Fixed
- `truncate_diff()` fallback for the single-huge-file case (the previous
  branch was unreachable and silently returned the full diff).

## [0.5.0] ā€” 2025-12-11

### Added
- `--review` / `--review-only` pre-commit review pass: the LLM scans the
  staged diff for likely defects and prints
  `[severity] file:line ā€” issue` findings.
- Review prompt forces `temperature=0.0` for reproducibility.
- `review.enabled = true` in config to always run the pass.

## [0.4.0] ā€” 2025-12-10

### Added
- `aicommit changelog <range>` subcommand.
- `--out CHANGELOG.md` prepend / replace of an existing `## Unreleased`
  block.
- Deterministic conventional-commit fast path ā€” when every commit in the
  range has a recognised prefix, no LLM call is made.
- Breaking-change (`feat!:` / `fix!:`) handling: routed into `Changed`.

## [0.3.0] ā€” 2025-12-08

### Added
- `git-aicommit` console script so `git aicommit` Just Works.
- Hierarchical TOML config: `defaults < ~/.config/aicommit/config.toml <
  <repo>/.aicommit.toml < AICOMMIT_* env < CLI flags`.
- argparse CLI flags: `--backend`, `--model`, `--url`, `--temperature`,
  `--max-tokens`, `--style`, `--no-body`, `--print`, `-y/--yes`,
  `--debug`, `--no-stream`.

## [0.2.0] ā€” 2025-12-05

### Added
- Interactive `[Enter / e / r / q]` loop after the proposal:
  - **Enter** runs `git commit -F <tmpfile>`.
  - **e** opens the message in `$EDITOR` (or `$VISUAL`, falling back to
    `vi`); an empty buffer aborts.
  - **r** regenerates, bumping temperature by `+0.15` each time.
  - **q** aborts (exit 130).
- Coloured `git diff --staged --stat` printed above the proposal,
  honouring `NO_COLOR` and isatty.

### Changed
- Refactored the single-file CLI into `prompts.py`, `git.py`,
  `llm/ollama.py`, and a thin `cli.py` glue layer.

## [0.1.0] ā€” 2025-12-02

### Added
- Initial CLI: reads the staged diff (stdin if piped, else
  `git diff --staged`), POSTs to a local Ollama instance with a
  hard-coded Conventional Commit prompt, prints the response on stdout.
- Environment-only config: `AICOMMIT_OLLAMA_URL`, `AICOMMIT_MODEL`,
  `AICOMMIT_TEMPERATURE`.
- Zero runtime dependencies ā€” stdlib `urllib` + `subprocess` only.
